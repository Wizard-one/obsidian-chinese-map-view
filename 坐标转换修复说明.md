# 坐标转换修复说明

## 问题描述

在使用高德地图源时，存在坐标转换不一致的问题：

1. **添加marker时**：坐标正确处理 - 从GCJ-02转换回WGS84保存
2. **编辑marker时（移动点）**：坐标未转换 - 直接保存GCJ-02坐标

## 原因分析

在`mapContainer.ts`文件中：

### 添加marker时的处理（正确）

```typescript
// 第710行左右 - 创建新marker时的处理
let markerLocation = e.layer.getLatLng();
// 如果使用高德地图，需要将GCJ-02坐标转换回WGS84保存
if (isAutoNaviMapSource(this.getMapSource())) {
    markerLocation = transformCoordinatesFromAutoNavi(markerLocation);
}
createMarkerInFile(markerLocation, ...);
```

### 编辑marker时的处理（错误）

```typescript
// 第1169行左右 - 编辑marker时的处理（修复前）
newMarker.on('pm:edit', (e) => {
    if (e.layer instanceof leaflet.Marker) {
        moveFileMarker(
            marker,
            e.layer.getLatLng(), // 直接使用了GCJ-02坐标，没有转换
            this.settings,
            this.app,
        );
    }
});
```

## 修复方案

在编辑marker时，同样需要进行坐标转换：

```typescript
// 修复后的代码
newMarker.on('pm:edit', (e) => {
    if (e.layer instanceof leaflet.Marker) {
        let newLocation = e.layer.getLatLng();
        // 如果使用高德地图，需要将GCJ-02坐标转换回WGS84保存
        if (isAutoNaviMapSource(this.getMapSource())) {
            newLocation = transformCoordinatesFromAutoNavi(newLocation);
        }
        moveFileMarker(marker, newLocation, this.settings, this.app);
    }
});
```

## 修复结果

现在添加marker和编辑marker时都会正确处理坐标转换：

- **添加marker**：GCJ-02 → WGS84 ✅
- **编辑marker**：GCJ-02 → WGS84 ✅（已修复）

这样确保了无论使用何种地图源，保存到文件中的坐标始终是标准的WGS84坐标系。
